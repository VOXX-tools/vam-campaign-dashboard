# データ収集の現状と課題

## 現在の実装

### ブラウザベースのデータ収集

現在、データ収集は**ブラウザが開いている時のみ**動作します：

1. **ユーザーがログイン**
2. **ブラウザが開いている間**、毎時5分に自動的にデータ取得
3. **Google Sheets API**からデータを取得
4. **Firestore**に保存
5. **ブラウザを閉じる**と、データ収集が停止

### 問題点

- **20時・21時のデータが欠けている理由**: その時間帯にブラウザが開いていなかった
- **データの欠損**: ブラウザを閉じている時間帯のデータが保存されない
- **チーム依存**: 誰かがブラウザを開いていないとデータが収集されない

## 解決策の選択肢

### オプション1: Vercel Cron Jobs（推奨だが有料）

**メリット**:
- サーバーサイドで確実に実行
- ブラウザ不要
- 管理が簡単

**デメリット**:
- Vercel Proプラン（$20/月）が必要
- Service Account認証の実装が必要

**実装状況**:
- ✅ `vercel.json`に設定追加済み
- ✅ `api/cron-fetch.ts`作成済み
- ❌ Service Account認証未実装

### オプション2: GitHub Actions（無料）

**メリット**:
- 完全無料
- GitHubで管理

**デメリット**:
- 設定がやや複雑
- GitHub Actionsの実行時間制限（月2,000分）

**実装状況**:
- ❌ 未実装

### オプション3: 現状維持（ブラウザベース）

**メリット**:
- 追加コストなし
- 実装済み

**デメリット**:
- データ欠損のリスク
- 誰かがブラウザを開いている必要がある

**運用方法**:
- オフィスのPCでブラウザを常時開いておく
- または、複数人が交代でブラウザを開く

## 推奨アクション

### 短期（今すぐできること）

1. **ブラウザを常時開いておく**:
   - オフィスのPCでダッシュボードを開きっぱなしにする
   - スリープモードにならないように設定

2. **複数人でカバー**:
   - チームメンバーが交代でブラウザを開く
   - 特に20時・21時をカバー

### 中期（1週間以内）

**GitHub Actionsを実装**（無料）:

1. Service Account認証を実装
2. GitHub Actionsワークフローを作成
3. テスト実行
4. 24時間監視

### 長期（予算がある場合）

**Vercel Proプランにアップグレード**:
- Vercel Cron Jobsを有効化
- より安定したデータ収集

## 次のステップ

どの方法を選択しますか？

1. **GitHub Actions（無料）を実装する** → Service Account認証の実装が必要
2. **Vercel Pro（$20/月）にアップグレードする** → Service Account認証の実装が必要
3. **現状維持（ブラウザベース）** → ブラウザを常時開いておく運用

選択肢を教えていただければ、実装を進めます。
